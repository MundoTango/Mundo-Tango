# VY PROMPT: APPLE & GOOGLE DEVELOPER ACCOUNT SETUP

**Agent:** Vy (Computer Access Agent via Vercept)  
**Objective:** Navigate logged-in Apple Developer and Google Play Console to retrieve account information  
**Timeline:** 30 minutes  
**Prerequisites:** User must have already created and paid for both developer accounts

---

## üéØ MISSION OVERVIEW

**IMPORTANT:** This task complies with Vercept TOS by working with platforms you're **already logged into**. Vy will NOT:
- ‚ùå Create new developer accounts
- ‚ùå Enter payment information ($99 Apple, $25 Google)
- ‚ùå Handle billing or credit card details

**User Must Complete First (Manually):**
1. Create Apple Developer account at https://developer.apple.com/programs/
2. Pay $99 enrollment fee (annual)
3. Create Google Play Developer account at https://play.google.com/console/signup
4. Pay $25 registration fee (one-time)
5. Stay logged into both consoles in your browser

**Vy's Role:** Navigate both logged-in consoles and retrieve account information  
**Replit Agent's Role:** Use account info to configure mobile app deployment

## üìã PHASE 1: APPLE DEVELOPER ACCOUNT VERIFICATION (10 minutes)

### Step 1.1: Verify User is Logged In
```
IMPORTANT: Before starting, user must be logged into https://developer.apple.com 
with an active Apple Developer Program membership ($99/year paid).

If not logged in:
- STOP and inform user to log in manually first
- Do NOT attempt to log in (violates Vercept TOS)
```

---

### Step 1.2: Navigate to Account Information
```
1. Navigate to: https://developer.apple.com/account
2. Confirm user is logged in (see account email in top-right)
3. Verify membership status:
   - Look for "Membership" section
   - Status should show "Active" or "Renewal due on [date]"
4. Note the following information:
   - Account Holder Name
   - Team ID (10-character alphanumeric, e.g., ABC123XYZ)
   - Organization Name (if organization account)
   - Membership Type (Individual or Organization)
   - Membership Expiration Date
```

---

### Step 1.3: Check Certificates
```
1. Navigate to: https://developer.apple.com/account/resources/certificates
2. Check if any certificates exist:
   - Development certificates
   - Distribution certificates
3. Note how many certificates exist
4. Do NOT create any certificates (Xcode will do this automatically)
```

---

### Step 1.4: Verify App IDs Section Access
```
1. Navigate to: https://developer.apple.com/account/resources/identifiers
2. Verify you can see the Identifiers page
3. Note if any App IDs already exist (likely none for new account)
4. Do NOT create App IDs yet (will be done during app build)
```

---

### Step 1.5: Compile Apple Account Information
```
Create a text file: apple_developer_info.txt

Apple Developer Account Information
====================================
Account Holder: [Name from account]
Team ID: [10-character ID]
Organization: [If applicable]
Membership Type: Individual / Organization
Membership Status: Active
Expiration Date: [Date]
Existing Certificates: [Count]
Account Email: [Email shown in top-right]

Notes:
- Membership is active and ready for iOS app deployment
- Xcode will automatically create necessary certificates and provisioning profiles
```

**Deliverable:** Apple account information (Team ID, status, etc.)

### Step 2.1: Verify User is Logged In
```

- STOP and inform user to log in manually first

```

---

### Step 2.2: Navigate to Account Information
```
1. Navigate to: https://play.google.com/console
2. Confirm user is logged in (see account name/email in top-right)
3. If you see "Create app" button, account is active
4. Click on account icon (top-right) ‚Üí "Account details"
5. Note the following information:
   - Developer Name
   - Developer Email
   - Developer ID
   - Account Type (Individual or Organization)
   - Registration Date
```

---

### Step 2.3: Check Play Console Setup
```
1. Click "Setup" in left sidebar (or "All apps" if setup complete)
2. Verify account setup is complete:
   - [ ] Developer account details
   - [ ] Payment profile (for paid apps - not applicable if only free apps)
   - [ ] Data safety form (will complete per app)
3. Note if any apps already exist (likely none for new account)
```

---

### Step 2.4: Verify API Access (Optional but Recommended)
```
1. Navigate to: https://play.google.com/console ‚Üí Settings ‚Üí Developer account ‚Üí API access
2. Check if Google Play Android Developer API is enabled
3. If disabled:
   - Click "Link" to link to Google Cloud Project
   - This allows automated uploads via Gradle (optional)
4. Note API status: Enabled / Not enabled
```

---
